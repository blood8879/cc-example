다음 요청사항을 바탕으로 GitHub 이슈를 생성해주세요:

$ARGUMENT

## 단계별 작업 수행:

### 1. 이슈 내용 분석 및 구조화
- 사용자가 제공한 내용을 분석하여 이슈의 핵심을 파악하세요
- 이슈 유형을 결정하세요 (버그, 기능 요청, 개선사항, 문서화, 질문 등)
- 프로젝트 컨텍스트를 고려하여 관련된 파일이나 컴포넌트를 식별하세요

### 2. 이슈 제목 작성
- 명확하고 간결한 제목을 작성하세요 (50-70자 이내)
- 이슈 유형을 제목에 반영하세요 (예: "버그: ", "기능: ", "개선: ")
- 구체적이고 검색 가능한 키워드를 포함하세요

### 3. 이슈 본문 작성
다음 템플릿을 사용하여 구조화된 이슈 본문을 작성하세요:

**버그인 경우:**
```markdown
## 🐛 버그 설명
[버그에 대한 명확하고 간결한 설명]

## 재현 방법
1. [첫 번째 단계]
2. [두 번째 단계]
3. [세 번째 단계]
4. [발생한 오류 확인]

## 예상 동작
[예상했던 정상 동작 설명]

## 실제 동작
[실제로 발생한 동작 설명]

## 환경
- OS: [예: macOS, Windows, Linux]
- 브라우저: [해당되는 경우]
- 버전: [애플리케이션 버전]

## 추가 컨텍스트
[스크린샷, 로그, 에러 메시지 등]
```

**기능 요청인 경우:**
```markdown
## 💡 기능 설명
[제안하는 기능에 대한 명확한 설명]

## 동기 및 배경
[이 기능이 왜 필요한지, 어떤 문제를 해결하는지 설명]

## 제안하는 해결 방안
[기능 구현 방법에 대한 제안]

## 대안
[고려한 다른 대안들]

## 추가 컨텍스트
[관련 스크린샷, 예시, 참고 자료 등]
```

**개선사항인 경우:**
```markdown
## 🔧 개선 제안
[개선하고자 하는 내용]

## 현재 상태
[현재의 동작이나 구현 방식]

## 제안하는 변경사항
[제안하는 개선 방법]

## 기대 효과
[개선으로 얻을 수 있는 이점]

## 추가 정보
[관련 정보나 참고 자료]
```

### 4. 라벨 선택
프로젝트에 적합한 라벨을 선택하세요:
- 이슈 유형: `bug`, `enhancement`, `feature`, `documentation`, `question`
- 우선순위: `priority: high`, `priority: medium`, `priority: low`
- 상태: `needs: triage`, `needs: investigation`, `good first issue`
- 영역: `frontend`, `backend`, `ui/ux`, `performance`, `security`

먼저 `gh label list`로 사용 가능한 라벨을 확인한 후 적절한 라벨을 선택하세요.

### 5. GitHub 이슈 생성
`gh issue create` 명령어를 사용하여 이슈를 생성하세요:
```bash
gh issue create \
  --title "제목" \
  --body "본문 내용" \
  --label "라벨1,라벨2"
```

### 6. 결과 보고
이슈 생성 후 다음 정보를 제공하세요:
- ✅ 생성된 이슈 번호
- 🔗 이슈 URL
- 🏷️ 적용된 라벨 목록
- 📝 작성된 이슈의 요약

## 주의사항:
- gh CLI가 설치되어 있고 인증이 완료되었는지 확인하세요
- 현재 디렉토리가 Git 저장소인지 확인하세요
- 원격 저장소가 GitHub에 있는지 확인하세요
- 사용자가 제공한 정보가 불충분한 경우, 추가 정보를 요청하세요
