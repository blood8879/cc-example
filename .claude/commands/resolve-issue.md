GitHub 이슈 #$ARGUMENT를 분석하고 해결하기 위한 상세한 계획을 수립해주세요.

## 단계별 작업 수행:

### 1. 이슈 정보 가져오기
`gh issue view $ARGUMENT`를 사용하여 이슈의 전체 내용을 가져오세요.
- 이슈 제목
- 이슈 본문
- 라벨
- 상태
- 코멘트 (있는 경우)

### 2. 코드베이스 컨텍스트 분석
이슈와 관련된 코드베이스를 분석하세요:
- 이슈에서 언급된 파일이나 컴포넌트를 찾아서 읽어보세요
- 관련 파일들의 구조와 의존성을 파악하세요
- 현재 구현 방식을 이해하세요
- 이슈가 버그인 경우, 버그가 발생할 수 있는 코드 위치를 특정하세요
- 이슈가 기능 요청인 경우, 기능을 구현할 적절한 위치를 파악하세요

필요한 경우 다음 도구들을 활용하세요:
- `Glob`으로 관련 파일 검색
- `Grep`으로 특정 코드 패턴 검색
- `Read`로 관련 파일 읽기

### 3. 문제 분석 및 근본 원인 파악
이슈를 깊이 있게 분석하세요:
- 문제의 근본 원인은 무엇인가?
- 왜 이 문제가 발생했는가?
- 어떤 엣지 케이스를 고려해야 하는가?
- 다른 부분에 영향을 줄 수 있는가?
- 비슷한 문제가 다른 곳에도 있을 수 있는가?

  - 장점:

  - 구현 복잡도:

- **방안 3**: [세 번째 접근 방법 (있는 경우)]

  - 단점:
  - 구현 복잡도:

각 방안의 트레이드오프를 명확히 설명하세요.

### 5. 추천 해결 방안 선정
최적의 해결 방안을 선택하고 그 이유를 설명하세요:
- 왜 이 방안이 가장 적합한가?
- 다른 방안들과 비교했을 때의 이점은?
- 잠재적 리스크는 무엇이며 어떻게 완화할 것인가?
