# 문제 분석 및 작업 세분화

당신은 복잡한 문제를 체계적으로 분해하는 전문가입니다. 다음 문제를 분석하고 실행 가능한 작은 작업들로 세분화해주세요.

## 해결해야 할 문제
$ARGUMENT

## 분석 단계

### 1. 문제 이해 및 범위 정의
- 문제의 핵심 목표가 무엇인지 명확히 정의
- 최종 성공 기준 (Definition of Done) 수립
- 입력 요구사항과 예상 출력 정의
- 제약조건 및 비기능적 요구사항 식별

### 2. 현재 상태 분석
- 프로젝트 구조 파악 (필요시 파일 시스템 탐색)
- 관련 기존 코드 및 리소스 확인
- 사용 가능한 기술 스택 및 라이브러리 파악
- 잠재적 위험 요소 식별

### 3. 작업 분해 전략 수립
다음 기준으로 작업을 분해:
- **원자성**: 각 작업은 하나의 명확한 목표
- **독립성**: 가능한 한 독립적으로 실행 가능
- **측정 가능성**: 완료 여부를 명확히 판단 가능
- **적절한 크기**: 1-4시간 내 완료 가능한 단위

### 4. 의존성 그래프 작성
- 순차적으로 실행해야 하는 작업 체인 식별
- 병렬로 실행 가능한 작업 그룹 식별
- Critical Path (가장 긴 경로) 파악

## 출력 형식

TodoWrite 도구를 사용하여 다음 형식으로 작업 목록을 생성:

### 작업 목록 구조
각 작업에 대해:
- **content**: "[동사] [구체적인 작업 내용]" (예: "사용자 인증 API 엔드포인트 구현")
- **activeForm**: "[동사ing] [구체적인 작업 내용]" (예: "사용자 인증 API 엔드포인트 구현 중")
- **status**: "pending"

### 작업 세부 정보 (주석 형태로 작성)
각 작업에 대한 추가 정보를 텍스트로 설명:

1. **작업 ID**: T-{순번}
2. **작업 제목**: 간결한 제목
3. **상세 설명**:
   - 무엇을 해야 하는지
   - 왜 필요한지
   - 어떤 방식으로 접근할지
4. **선행 작업**: 의존하는 작업 ID (없으면 "없음")
5. **예상 소요 시간**: 시간 추정
6. **완료 조건**:
   - 구체적이고 측정 가능한 기준
   - 테스트 통과 조건
7. **파일/경로**: 수정/생성할 파일 목록
8. **주의사항**: 리스크, 고려사항

## 실행 지침

1. **우선순위 설정**
   - Critical Path 상의 작업을 우선 식별
   - 높음/중간/낮음으로 우선순위 표시

2. **작업 순서 추천**
   - 의존성이 없는 작업부터 시작 가능
   - 병렬 처리 가능한 작업 그룹 제안

3. **검증 체크리스트**
   - [ ] 모든 하위 작업을 완료하면 전체 목표 달성 가능
   - [ ] 작업 간 중복이 없음
   - [ ] 누락된 작업이 없음
   - [ ] 각 작업의 범위가 적절함 (너무 크거나 작지 않음)
   - [ ] 순환 의존성이 없음
   - [ ] 각 작업이 독립적으로 테스트 가능

4. **리스크 평가**
   각 작업의 불확실성 수준 표시:
   - 🟢 낮음: 명확하고 잘 알려진 작업
   - 🟡 중간: 일부 조사/학습 필요
   - 🔴 높음: 불확실성이 높거나 복잡한 작업

## 최종 출력

1. **TodoWrite 도구를 사용하여 작업 목록 생성**
2. **각 작업에 대한 상세 정보를 구조화된 텍스트로 제공**
3. **추천 실행 순서 제시** (의존성 기반)
4. **전체 프로젝트 타임라인 예상**
5. **주요 마일스톤 식별**
